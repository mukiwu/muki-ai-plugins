# 階段 0：需求釐清（蘇格拉底式提問）

在規劃之前，先透過提問釐清需求：

- **一次只問一個問題**，不要連珠炮
- **優先用選擇題**勝於開放式問題（例如「A 還是 B？」而非「你想怎麼做？」）
- 探索 **2-3 個不同方案**，說明各方案的取捨
- 設計方案分段確認：「這樣對嗎？」
- 積極套用 **YAGNI 原則**：設計階段就砍掉不必要的功能

> 跳過條件：使用者的需求已經非常明確具體（例如「在 X 檔案的 Y 函式加入 Z 邏輯」）。

此階段不需要暫停，釐清完直接進入階段 1。
